### 3. Git版本管理规范已建立

**Git Flow工作流**:
```
main (生产) ← release (预发布) ← develop (开发)
                     ↑
              feature/xxx (功能分支)
```

**分支规范**:
| 分支类型 | 命名 | 用途 |
|----------|------|------|
| main | main | 生产环境代码 |
| develop | develop | 开发主分支 |
| feature | feature/功能名 | 新功能开发 |
| release | release/v版本号 | 版本发布准备 |
| hotfix | hotfix/问题描述 | 紧急修复 |

**提交规范**:
```
<type>(<scope>): <subject>

type: feat/fix/docs/style/refactor/test/chore
```
